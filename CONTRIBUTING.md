## Contribution Process

1. **Fork** the repo
2. **Create** a branch (`feature/my-contribution`)
3. **Add** your content in the appropriate directory
4. **Test** your prompts/workflows actually work
5. **Submit** a pull request

## Style Guide

### Markdown

- Use headers for organization
- Code blocks with language tags
- Horizontal rules between major sections
- Keep lines under 100 characters

### Prompts

```
[Brief description of when to use]

[The actual prompt template]

[Example usage]
```

### Conversations

Use this format for example conversations:

```
### User

[User prompt]

### Claude

[Claude response]
```

## Quality Standards

Before submitting:

- [ ] Prompts have been tested and work reliably
- [ ] Examples are realistic and useful
- [ ] Writing is clear and concise
- [ ] No proprietary or sensitive information
- [ ] Follows existing structure and style

## What NOT to Submit

- Prompts for harmful purposes
- Content that requires specific API keys or paid services
- Outdated information (check your examples work!)
- Self-promotional content
- Duplicate content

## Questions?

Open an issue if you're unsure about something. We're happy to help!
